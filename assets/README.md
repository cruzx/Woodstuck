# assets

`assets/` 是 `woodstuck` 的视觉证据层，不是一个随手堆图的临时目录。

这里存两类东西：

- 历史协作里已经被引用过、暂时不适合迁移路径的共享截图
- 已经按主题沉淀好的资料包与截图池

## 当前分层

- `assets/竞品/`
  - 竞品与灵感截图池
  - 按 `发现与分发 / 播放与沉浸 / 免费模式 / 会员转化 / 活动运营 / 规模扩张` 这类可复用玩法结构归档
- `assets/会员特权/`
  - 会员特权资料包入口
  - 用来承接概念版会员、音质、下载、铃声、权益表达等任务的源材料
- `assets/` 根目录
  - 只保留历史共享图、跨主题证据图、暂未迁移但已有引用的文件

## 使用规则

1. 新增截图默认不要直接丢在 `assets/` 根目录。
2. 竞品、页面灵感、分类截图，优先放进 `assets/竞品/` 对应桶。
3. 会员权益、特权清单、权益说明类资料，优先放进 `assets/会员特权/`。
4. 如果一批新图还没来得及整理，先放进 `assets/竞品/00-待归类/`，不要在根目录平铺。
5. 如果某张旧图已在文档里被路径引用，不要为了整理结构直接改路径；先补 README 说明，再择机迁移。

## 维护原则

- `assets/` 负责存视觉证据，不负责写机制判断
- 机制结论、复用规则、趋势判断，写回 `learning/`、`references/`、`playbooks/`
- 如果一个目录已经需要解释“为什么这么分”，就补 README，而不是继续靠口头记忆
